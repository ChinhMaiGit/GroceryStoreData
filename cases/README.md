# Cases — the business briefs that give each dataset its story

One case per scenario arm, bound by folder name: `cases/<arm>/` pairs with
`data/scenarios/<arm>/`. Each case folder holds exactly two files, both
carrying frontmatter that names the arm and its data path:

```
cases/<arm>/BUSINESS_CASE.md              <- the student packet (hand out with visible/)
cases/<arm>/BUSINESS_CASE_INSTRUCTOR.md   <- grading map, traps, reveal staging (never hand out)
```

The cases are narrative, not rules: how to *write* one — the epistemic
firewall, the persona and confession devices, the QA sweep — is specified
in `documents/CASE_WRITING_GUIDE.md`. Brand and persona (Henrik Malm,
Malm's Market, Lindåker) stay constant across arms: the same shop living
different histories, so a new scenario needs only its events re-told in
the owner's voice and its own wrong belief chosen.
